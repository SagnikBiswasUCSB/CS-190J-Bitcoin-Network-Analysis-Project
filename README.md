# CS190J Bitcoin Project: Unveiling Market Structures through Advanced Network Analysis, Temporal Dynamics, and Topological Signatures

**Authors:** Pratyush Rallapally, Matthew Zhang, Rishik Buneti, Sagnik Biswas

## Project Overview

This project investigates advanced network analysis techniques to identify and characterize meaningful community structures within Bitcoin trust-based ecosystems. Traditional methods often provide static or incomplete views, struggling with signed relationships, crucial higher-order interactions, and the vital **temporal dynamics** inherent in these networks. This research aims to:

1.  Systematically evaluate and apply methodologies including:
    * Nuanced modularity optimization (e.g., Louvain for community detection).
    * Spectral analysis via Graph Laplacians ($L_0$) for community partitioning (Fiedler vector) and structural (unweighted incidence) Hodge Laplacians ($L_0, L_1, L_2$) to capture multi-scale coordination patterns and topological features (related to Betti numbers $\beta_0, \beta_1, \beta_2$).
    * Simulated Topological Data Analysis (TDA), with a strong emphasis on **time-based filtrations (using edge timestamps to define an evolving clique complex)** alongside conceptual attribute-based filtrations, to illustrate the identification of evolving topological features (components H0, cycles H1).
2.  Apply these techniques to real-world Bitcoin user rating datasets (Bitcoin OTC and Bitcoin Alpha), which contain rich, timestamped interaction data.
3.  Attempt to decode the fundamental "operational blueprints" and "topological fingerprints" of implicit network communities, particularly focusing on their **temporal behavior, stability, and evolution**.
4.  Pinpoint which advanced methodologies, especially those leveraging temporal information and higher-order network structure, offer the clearest and most interpretable insights into these complex digital economies.

The goal is to move beyond surface-level clustering to reveal persistent and evolving structural patterns that characterize different market actors and their operations over time, providing a richer understanding of network organization.

## Methodology Implemented/Simulated

The `script.py` implements or simulates the following analyses (targeting 5-7 advanced approaches):

* **Community Detection:**
    * Louvain Modularity optimization (handles weights, adapted for signed nature by using absolute values for partitioning).
    * Girvan-Newman divisive clustering (modularity-tracked, for smaller graphs/components).
    * k-Clique Percolation for overlapping communities (e.g., k=3, k=4).
* **Spectral Methods & Hodge Laplacians:**
    * $L_0$ (Graph Laplacian from $B_1 B_1^T$): Fiedler vector partitioning for spectral bisection; analysis of $\lambda_2$.
    * Structural (Unweighted Incidence) Boundary Operators: $B_1$ (node-to-edge) and $B_2$ (edge-to-triangle).
    * Structural Hodge Laplacians: $L_0=B_1B_1^T$, $L_1^{\text{down}}=B_1^TB_1$, $L_1^{\text{up}}=B_2B_2^T$, $L_1^{\text{full}}=L_1^{\text{down}}+L_1^{\text{up}}$, and $L_2^{\text{down}}=B_2^TB_2$.
    * Spectral analysis of $L_0, L_1, L_2$ to infer information about $\beta_0, \beta_1, \beta_2$ (connected components, cycles, voids).
* **Simulated Topological Data Analysis (TDA):**
    * Framework for simulating a **Vietoris-Rips style filtration using edge timestamps (time $t$ as $\epsilon$)** to build an evolving clique complex.
    * **Illustrative generation of persistence barcodes (simulated H0, H1)** based on the evolution of connected components and heuristic cycle detection within the time-filtered graph snapshots. This simulates the *type* of output and insight expected from a full TDA library.
* **Temporal Analysis:**
    * Direct use of timestamps in the simulated TDA time filtrations is a core component.
    * The framework supports creating network snapshots, enabling other methods (e.g., community detection, Hodge Laplacians) to be applied dynamically, although the main script focuses on full graph analysis for some methods and TDA for explicit time evolution.

## Important Notes on Advanced Method Simulations

* **Hodge Laplacians:** The $B_1$ and $B_2$ boundary operators, and consequently the Hodge Laplacians ($L_0, L_1, L_2$), are constructed based on the *unweighted, structural* incidence of nodes, edges, and 3-cliques (triangles). Rigorous algebraic topology for weighted or consistently oriented *directed* Hodge Laplacians is more complex and typically handled by specialized libraries.
* **Topological Data Analysis (TDA):** The persistent homology barcodes (H0, H1) generated by `script.py` are **illustrative simulations**. They are based on graph component evolution (for H0) and heuristic cycle detection (for H1) within a time-based filtration. True persistent homology computation for higher dimensions or with mathematical rigor requires dedicated algorithms and libraries (e.g., GUDHI, Ripser, Dionysus) that operate on explicitly constructed filtered simplicial complexes. This script aims to demonstrate the *concept and potential utility* of time-based TDA.

The goal is to provide a runnable script that explores these advanced concepts as deeply as feasible with standard scientific Python packages, emphasizing the interpretation of their outputs for network understanding.
